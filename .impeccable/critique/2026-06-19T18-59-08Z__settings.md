### Overall Impression

Solid bones — tab navigation, toggle switch, and destructive delete pattern all well-executed. Two immediate issues: Save buttons missing lft-btn base class (every primary action is visually broken), and no unsaved-change guard means silent data loss on tab switch.

### Strengths

1. Toggle switch — role="switch" + aria-checked + correct translate-x + focus-visible ring. Best-implemented component on the page.
2. Destructive delete pattern — DELETE-confirmation gate, disabled button, red-tinted danger zone card, sr-only hint for screen readers.
3. Password show/hide — all three fields have independent toggles with dynamic aria-label and aria-hidden SVGs.

### Priority Issues

P0 - Save buttons missing lft-btn base class: lft-btn-primary requires lft-btn for height (44px), border-radius, font-weight, transitions, focus ring, active:scale. Without it buttons have no height guarantee, no focus ring, no press animation. Fix: add lft-btn to every Save/Change button; use lft-btn lft-btn-danger for delete.

P1 - No unsaved-change guard: clicking any tab while a form is dirty silently discards all changes. Fix: track isDirty signal per section, show confirmation dialog in setSection().

P1 - Section h2 eyebrows redundant with active tab: "Account Settings" h2 next to the active "Account" tab adds triple heading hierarchy with no gain. Fix: remove eyebrows or make them sr-only.

P2 - Card backgrounds opaque bg-zinc-900 instead of lft-card: loses the translucent depth effect used everywhere else. Fix: replace with lft-card class.

P2 - Password form not submittable via Enter: inputs are in a div not a form element. Fix: wrap in form with ngSubmit.

P3 - Appearance tab too thin for a dedicated tab: two radio buttons don't justify the context switch. Fix: merge into Account or Workout as a preference row.
