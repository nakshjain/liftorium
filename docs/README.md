# Gym Helper Documentation

This directory contains the working documentation for Gym Helper, a mobile-first workout tracking application built with Angular, Node.js, Express, MongoDB, and JWT authentication.

## Documentation Map

| Area | Purpose |
| --- | --- |
| [Architecture](./architecture/README.md) | System design, frontend/backend structure, data model, and technical decisions |
| [API](./api/README.md) | REST API contracts, authentication flows, and endpoint specifications |
| [Workflows](./workflows/README.md) | Development, testing, release, and AI-assisted engineering workflows |
| [Prompts](./prompts/README.md) | Prompt history, implementation prompts, review prompts, and reusable AI instructions |
| [Progress](./progress/README.md) | MVP roadmap, implementation log, status tracking, and open work |
| [Decisions](./decisions/README.md) | Architecture decision records and rationale |

## Documentation Principles

- Keep documentation close to implementation changes.
- Update API docs whenever request or response contracts change.
- Record architecture decisions before or during meaningful structural changes.
- Save important AI prompts that shape implementation direction.
- Keep progress logs short, dated, and actionable.

## MVP Priority

## Maintenance Checklist

Use this checklist after each feature implementation:

- [ ] Update relevant architecture notes.
- [ ] Update API contracts.
- [ ] Add or update workflow documentation.
- [ ] Save important prompts in prompt history.
- [ ] Add progress log entry.
- [ ] Add architecture decision record if a significant technical choice was made.
