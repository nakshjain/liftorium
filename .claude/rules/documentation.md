# Documentation

After implementing a feature, update the relevant files under `/docs`:
- `/docs/api/` — endpoint contracts (request/response shapes, auth requirements).
- `/docs/architecture/` — structural changes (new services, data model changes).
- `/docs/decisions/` — new ADRs when making significant technology choices.
- `/docs/progress/progress-log.md` — what was done and what's next.
